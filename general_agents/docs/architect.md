# Architect Agent

## What it does
The architect designs the structure of a system before any code is written. It thinks in terms of components, data flow, file/folder layout, and technology choices. It outputs a written plan — not implementation code.

## When to use it
- Before starting a new project or major feature
- When you're unsure how to structure a codebase
- When you need to choose between different technical approaches

## Tools it can use
| Tool | Why |
|------|-----|
| Read | Review existing code if extending a project |
| Glob | Understand current file structure |
| Grep | Find patterns in existing code |
| WebSearch | Research best practices and available tools |

## Example task
```
Design the architecture for a real-time chat app using Python. Include component breakdown, file structure, and technology recommendations.
```

## Typical workflow position
**Second, after the researcher.** Produces a plan that the developer then implements.

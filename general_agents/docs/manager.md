# Manager Agent

## What it does
The manager oversees a project at a high level. It breaks down tasks, decides which specialist should handle what, and ensures work is progressing coherently. It does **not** write code itself.

## When to use it
- At the start of a new project or feature to create a plan
- When a task is too broad and needs to be broken into smaller pieces
- When you need a bird's-eye view of what work remains

## Tools it can use
| Tool | Why |
|------|-----|
| Read | Review existing code and files |
| Glob | Understand project file structure |
| Grep | Search for specific patterns across the codebase |
| Bash | Run commands to check project state |

## Example task
```
Review the project structure and create a step-by-step plan for adding user authentication.
```

## Typical workflow position
**First.** The manager usually starts a workflow, then hands off subtasks to other agents.
